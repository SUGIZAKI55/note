# Gitリポジトリ登録方法（20260201）
  
## 1. 習得した主要コマンド  
| コマンド | 意味 |  
| :--- | :--- |  
| `git init` | Gitリポジトリの初期化（`.git`フォルダの作成） |  
| `ls -al` | 隠しファイル（`.git`等）を含む全ファイルの詳細表示 |  
| `git status` | 現在の変更状況やステージング状態の確認 |  
| `git add .` | 全ての変更をステージングエリア（保存対象）に追加 |  
| `git commit -m "msg"` | メッセージを付けてローカルに履歴を保存 |  
| `git remote add origin [URL]` | リモートリポジトリ（GitHub）の登録 |  
| `git remote -v` | 登録されているリモートURLの一覧表示 |  
| `git push -u origin main` | 初回プッシュと上流ブランチ（紐付け）の設定 |  
| `rm -rf [dir]` | ディレクトリを中身ごと強制削除（リセット用） |  
  
---  
  
## 2. 実験と検証の記録  
  
### 実験①：test リポジトリの作成  
- 目的：最初のGitHub連携テスト。  
- 結果：成功。`index.html`をGitHubへPush。  
  
### 実験②：test2 での重複エラー検証  
- 現象：`error: remote origin already exists.` が発生。  
- 原因：同じディレクトリ内に、既に過去の`origin`設定が残っていたため。  
- 学び：  
    - `git remote add` は「新規追加」用。  
    - 既存のURLを書き換えるなら `git remote set-url origin [URL]` を使うのがプロの定石。  
    - 設定はフォルダ（`.git`）ごとに独立して管理されている。  
  
### 実験③：test3 でのクリーンな構築  
- 手順：新規フォルダ作成 → `git init` → `README.md`作成 → `commit` → `push`。  
- 結果：成功。一番綺麗な手順で複数リポジトリ管理を完遂。

## 3. 発生したトラブルと解決策  
- **コマンドミス**: `remote add` の前に `git` を忘れると `-bash: remote: command not found` になる。  
- **Mac特有の挙動**: `git add .` をすると `.DS_Store` というシステムファイルも入ってしまう（今後は `.gitignore` で対策が必要）。  
- **ユーザー名設定**: 初回コミット時にユーザー情報の自動設定に関する案内が出る。`git config --global` で設定可能。
